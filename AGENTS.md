# 项目规则

## 1. 代码设计
- 代码设计要清晰明了，保持良好的结构和可读性
- 遵循项目现有的代码风格和架构模式

## 2. 执行原则
- 不要随意幻想或臆测，严格按照用户的要求执行
- 在拿不准的地方要及时询问用户，确认后再继续
- 宁可多问一句，也不要自作主张导致返工

## 3. 中文注释
- 代码中要有清晰合理的中文注释
- 注释应解释"为什么"而非"做了什么"，帮助后续维护者理解设计意图
- 函数、模块的关键逻辑处应添加必要的中文说明

## 4. 测试规范
- 所有测试必须使用本项目内的虚拟环境执行
- 测试代码和测试数据必须放在本项目文件夹内，不要放到用户主文件夹或其他位置
- 测试完成后及时清理临时测试数据

## 5. 文档编码
- 所有文档默认使用 UTF-8 编码
- 在 PowerShell 中阅读文档时，必须先声明使用 UTF-8 编码（如 `chcp 65001` 或 `[Console]::OutputEncoding = [System.Text.Encoding]::UTF8`）

## 6. 沟通反馈
- 及时总结已完成的工作和当前进展
- 遇到问题、错误或阻塞时及时向用户反馈
- 完成关键步骤后主动汇报结果
